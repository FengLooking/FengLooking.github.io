1.DNS解析

DNS：Domain Name System，域名系统的缩写。
通过主机名、最终得到主机名对应的ip地址的过程叫做域名解析(或主机名解析)
过程如下: 

1.用户主机上运行有DNS客户端  
2.浏览器接收到URL后，将URL中的域名字段解析出来，并将该域名字段传送给DNS应用的客户端  
3.DNS客户端向DNS服务器端发送一份查询报文，报文中包含要访问的域名字段  
4.DNS服务器端返回一份应答报文给DNS客户端，应答报文中包含域名对应的ip地址  
5.浏览器收到来自DNS的ip地址后，向该ip地址对应的HTTP服务器发起TCP连接
浏览器是如何通过域名去查询对应的IP(对应的服务器地址)的？
解析顺序如下：其中1、2、3称为客户端DNS缓存 
1.浏览器缓存：当用户通过浏览器访问某域名时，浏览器首先会在自己的缓存中查找是否有该域名对应的IP地址。如果浏览器访问过该域名且没有清理缓存的情况下IP信息是存在的；  
2.系统缓存：当浏览器缓存中没有域名对应的IP，则会自动检查用户计算机系统Host文件DNS缓存中是否存在该域名对应的IP地址；  
3.路由器缓存：当浏览器及系统缓存中都没有域名对应的IP则到路由器缓存中进行查询；  
4.ISP DNS缓存：在用户客户端查询不到域名对应的IP地址时，就会进入ISP(Internet Service Provider)互联网服务提供商 DNS缓存中进行查询；  
5.根域名服务器：当以上均未完成，则进入根域名服务器查询。根域名服务器收到请求后会查询区域文件记录，若无则将其管辖范围内的顶级域名(如.com)服务器IP告诉本地DNS服务器；  
6.顶级域名服务器：顶级域名服务器收到请求后查看区域文件记录，若无则将其管辖范围的主域名服务器的IP地址告诉本地DNS服务器；  
7.主域名服务器：主域名服务器收到请求后查询自己的缓存，若无则进入下一级域名服务器进行查询，并且会重复该步骤直到查询到正确记录；  
8.保存结果至缓存：本地域名服务器把返回的结果保存到缓存，以备下一次使用，同时将结果反馈给客户端，客户端会通过这个IP地址与WEB服务器建立链接。

2.docker与虚拟化的区别

   虚拟化技术是一个总称，是一系列实现虚拟技术的统称。虚拟化技术包括了虚拟机技术和容器技术。虚拟化技术最大的特点就是将一个真实的机器进行虚拟地分割，然后分割出来的部分可以独立使用。
   虚拟机(Virtual Machine)是指通过软件模拟的具有完整硬件系统功能的、运行在一个完全隔离环境中的完整计算机系统。即：把一个硬件划分为多个机器，每个机器都能独立运行。
   容器实际也是采用虚拟的方式进行的，容器技术也能实现在一台硬件机器上虚拟出多个容器，每个容器中都可以放置一个互相隔离的服务器。在同等条件下，容器技术比虚拟机技术效率高，但是容器中没有包含独立的操作系统，虚拟机中包含独立的操作系统，所以每台虚拟机中的功能比每个容器的功能要多，正是因为虚拟机带有的东西多，所以占用的系统资源大。

3.负载均衡四层和七层

   负载均衡是建立在现有网络结构之上，提供了一种廉价有效的方法扩展网络设备和服务器的带宽、增加吞吐量、加强网络数据处理能力、提高网路的灵活性和可用性。  
   负载均衡(Load Balance)就是把任务分摊到多个操作单元上进行执行，如：WEB服务器，FTP服务器等，从而完成工作任务。  
   负载均衡四层就是IP+端口的负载均衡，通过虚拟IP+端口接收请求，然后再分配到真实的服务器，主要是通过报文中的目标地址和端口，再加上负载均衡设备设置的服务器选择方式，决定最终选择的服务器。
   负载均衡七层就是基于URL等应用层信息的负载均衡，通过虚拟的URL或者主机名接收请求，然后再分配到真实的服务器，七层负载均衡也称为内容交换：就是主要通过报文中真正有意义的应用层内容，加上负载均衡设备设置的服务器选择方式，决定最终选择的服务器。

8.tar、gzip、zip的区别

打包和压缩。打包是指将一大堆文件或目录变成一个总的文件；压缩则是将一个大的文件通过一些压缩算法变成一个小文件。
tar将多个文件或目录打包在一起，可用通过调用gzip或zip实现压缩、解压的命令；tar不仅可以多多个文件进行打包，还可以对多个文件打包后进行压缩。
gzip命令的作用是将一个大的文件通过压缩算法，变成一个小的文件，gzip文件不能直接压缩目录，如果要压缩目录，需要先将目录打包成一个文件，在压缩。
zip是linux和windows等多平台通用的压缩格式。zip比gzip更强的是zip命令压缩文件不会删除源文件，还可以压缩目录。

zip包  
压缩： zip -r [目标文件名].zip [原文件/目录名]  
解压： unzip [原文件名].zip  
注：-r参数代表递归

tar包 仅仅打包，不压缩  
打包：tar -cvf [目标文件名].tar [原文件名/目录名]  
解包：tar -xvf [原文件名].tar  
注：c参数代表create（创建），x参数代表extract（解包），v参数代表verbose（详细信息），f参数代表filename（文件名），所以f后必须接文件名。

tar.gz格式  
方式一：利用前面已经打包好的tar文件，直接用压缩命令。
压缩：gzip [原文件名].tar  
解压：gunzip [原文件名].tar.gz  

方式二：一次性打包并压缩、解压并解包
打包并压缩： tar -zcvf [目标文件名].tar.gz [原文件名/目录名]  
解压并解包： tar -zxvf [原文件名].tar.gz  
注：z代表用gzip算法来压缩/解压。

9.linux用户组权限管理

# 查看当前目录（包含隐藏文件）的长格式。
ls -la  
  -rw-r--r--   1 user  staff   651 Oct 12 12:53 .gitmodules
d ：第一位表示文件类型  
　　d 文件夹  
　　- 普通文件  
　　l 链接  
　　b 块设备文件  
　　p 管道文件  
　　c 字符设备文件  
　　s 套接口文件

rwx ：第2-4位表示这个文件的属主(当前用户)拥有的权限。r是读、w是写、x是执行
r-x ：第5-7位表示和这个文件属主所在同一个组的用户(组用户)所具有的权限
r-x ：第8-10位表示其他用户所具有的权限

通常用三个数字来表示文件的读取、写入、执行权限：
读取：4
写入：2
执行：1

例如；
drwxr-xr-w
rwx 4+2+1 7 所有权限
r-x 4+1 5 读和执行权限，没有写权限
r-w 4+2 6 读写权限，没有执行权限
具体使用：  
chmod 766 文件名

改变文件的owner  
chown 用户:用户组 文件名  
例如：chown tctadmin:tctgroup test.xls

10.linux路由配置

# 添加到主机的路由
route add -host 192.168.1.2 dev eth0
route add -host 10.20.30.148 gw 10.20.30.40
# 添加到网络的路由
route add -net 10.20.30.40 netmask 255.255.255.248 eth0
route add -net 10.20.30.48 netmask 255.255.255.248 gw 10.20.30.41
route add -net 192.168.1.0/24 eth1
# 添加默认路由
route add default gw 192.168.1.1  
# 删除路由
route del -host 192.168.1.2 dev eth0
